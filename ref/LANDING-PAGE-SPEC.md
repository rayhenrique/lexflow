# Especificação Estrutural da Landing Page

**Objetivo:** Uma página (single-page) focada em conversão para escritórios de advocacia que buscam organização financeira segura.

## Estrutura das Seções:

1. **Hero Section (Topo)**
   - **Objetivo:** Capturar atenção imediata com o benefício principal.
   - **Layout:** Centralizado. Texto grande + Subtexto + CTA primário (Ex: "Agendar Demonstração").
   - **Visual:** Abaixo do CTA, um mock-up do sistema (Dashboard com o Context Switcher em destaque) com cantos arredondados e sombra suave.

2. **Problem/Agitation**
   - **Objetivo:** Gerar identificação com a dor do gestor.
   - **Layout:** Grid de 3 colunas simples.
   - **Visual:** Ícones de alerta (lucro oculto, mistura de contas, ferramentas ruins).

3. **Solution / The "Workspace" Concept (Destaque)**
   - **Objetivo:** Explicar a separação por áreas de atuação.
   - **Layout:** Z-pattern (Texto na esquerda, Imagem/GIF na direita).
   - **Visual:** Mostrar um pequeno GIF/vídeo de 3 segundos do usuário trocando de "Trabalhista" para "Matriz" e os gráficos mudando instantaneamente.

4. **Features Grid**
   - **Objetivo:** Apresentar as funcionalidades secundárias.
   - **Layout:** Grid de cards (2x2 ou 3x2).
   - **Conteúdo dos cards:** Segurança (RLS explícito para os mais técnicos), Dashboards automáticos, Exportação de relatórios, Controle de inadimplência.

5. **Social Proof (Validação)**
   - **Objetivo:** Construir confiança.
   - **Layout:** Centralizado, com o logo/depoimento do cliente âncora.

6. **Footer / Final CTA**
   - **Objetivo:** Última chance de conversão.
   - **Layout:** Bloco escuro (para contraste, mesmo sendo light mode no produto), CTA limpo.
